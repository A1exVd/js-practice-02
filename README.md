### Вопросы к заданию

**В чём разница между == и ===? Приведите пример, где == может дать неожиданный результат**
При использовании нестрогого равенства (==) происходит неявное приведение типов, а потом сравнение значений. А при использовании строгого - сравнение по типу и значению. Типичной ошибкой может быть обработка числа 0, где значение 0, введенное пользователем, и пустая строка обрабатываются одинаково. Или обработка значений null и undefined.

**Что такое приведение типов (type coercion)? Что получится в выражениях: 5 + "5" и 5 - "5"?**
Преобразование значений из одного типа в другой. Может быть явным и неявным.  
5 + "5" = "55"  
5 - "5" = 0

**Когда уместно использовать switch, а когда лучше if/else?**
Конструкции switch и if/else управляют ветвлением кода (Выполнение разных блоков кода в зависимости от истиности заданного условия).
Switch лучше использовать, когда есть большое количество условий (больше 3) для переменной или выражения. Например, день недели, для каждого дня недели выполняем определенный блок кода.
if/else - при проверке сложных условий или диапазонов или если вариантов ветвлений мало.

**Что вернет prompt(), если пользователь нажмет "Отмена"? Как вы это обрабатываете в своем квизе?**
prompt() при отмене вернет null.  
if (answer === null) {  
    // ипсользуем Early return здесь  
    return  
}

**Как проверить, что в переменной действительно число, а не NaN?**
Используя метод Number.isNaN(), который вернет true, если значение является NaN. При вызове метода передаем в него Number(value) в качестве аргумента.

**Что такое Early Return и чем он лучше вложенных if/else?**
Early return досрочное выполнение функции при достижении определенного условия. Позволяет уменьшить вложенность, количество else и улучшить читаемость.

**Зачем нужен блок default в операторе switch?**
default используется для выполнение блока кода, если ни один из варинтов case не сработал. Если опустить default, то switch может выполниться без результата, что может привести к наличию скрытых багов или ошибок.
